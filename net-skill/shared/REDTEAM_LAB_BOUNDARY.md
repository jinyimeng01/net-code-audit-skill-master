# 授权红队实验室边界

本 skill 集合可用于攻防一体化代码审计，但默认行为必须是授权、安全、可复核。

## 使用边界

- 仅在用户明确拥有授权的源码、测试环境、靶场、内网实验室或客户授权范围内使用。
- 默认输出无害验证证据，不默认生成可直接投递到真实第三方系统的武器化 payload。
- 涉及命令执行、反序列化、SSRF、凭据伪造、ViewState/machineKey、横向移动的内容，只能放在“授权实验室验证建议”小节。
- 不要输出真实密钥、完整生产连接串、可复用 token；报告中只保留脱敏片段或哈希。

## PoC 规则

| 类型 | 默认验证 |
|---|---|
| 命令执行 | 固定字符串、临时文件、低权限命令、日志证据 |
| SSRF | 授权 callback 域名、本地 mock、禁止默认访问真实 metadata |
| 反序列化 | 类型控制与 sink 可达性证明、隔离环境无害命令 |
| 文件读取 | 读取测试文件或非敏感固定文件，不泄露生产敏感文件 |
| 鉴权/越权 | 使用两个授权测试账号或构造化权限模型证明 |

## 报告要求

- 明确标注授权范围、测试环境、验证强度、残余风险。
- 对无法安全验证的高危链路，给出代码证据和实验室复现步骤，不给生产攻击步骤。
- 修复建议优先于利用细节。
