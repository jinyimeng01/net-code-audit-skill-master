# .NET 审计风险分级标准

所有 .NET 审计 skill 统一使用 `Critical / High / Medium / Low / Info`。

## 判定维度

| 维度 | 说明 |
|---|---|
| 可达性 | 是否可从 HTTP 路由、WCF/SOAP、队列、文件导入、后台任务等外部入口到达 |
| 鉴权状态 | 未认证、普通用户、低权限后台用户、管理员、仅本地 |
| 输入控制 | 用户是否能控制 sink 的关键参数、类型、路径、URL、命令、SQL、XML |
| 影响 | RCE、认证绕过、敏感数据泄露、业务越权、DoS、信息泄露 |
| 利用复杂度 | 是否需要密钥泄露、竞态、特殊组件版本、内网条件、用户交互 |
| 证据强度 | 代码证据、调用链、无害 PoC、实验室复现、仅模式匹配 |

## 等级

| 等级 | 标准 |
|---|---|
| Critical | 未认证或低权限可达 RCE、任意文件读写、认证伪造、关键密钥泄露可直接接管系统 |
| High | 认证用户可达高影响漏洞；或需要一个明确前置条件即可达到 RCE/敏感数据泄露 |
| Medium | 影响受限、需要较高权限或环境条件；可造成局部越权、信息泄露、有限 SSRF/DoS |
| Low | 配置不佳、弱防护、版本过旧但未发现触发入口，或仅内部低影响问题 |
| Info | 架构观察、加固建议、不可利用但值得记录的安全上下文 |

## 证据标签

- `Confirmed`: 有入口、调用链、sink、无害验证或实验室复现。
- `Reachable`: 有入口到 sink 的代码路径，但未执行动态验证。
- `Potential`: 关键模式存在，但输入可控性或环境条件未确认。
- `Not exploitable`: 存在危险 API，但有强约束、不可达或固定输入。

报告中不要只写等级，必须同时写证据标签。
