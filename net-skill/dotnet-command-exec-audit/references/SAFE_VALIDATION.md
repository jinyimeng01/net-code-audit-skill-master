# 命令执行无害验证模板

## 默认原则

- 不在生产系统执行破坏性命令。
- 优先用代码证据、调用链和受控实验室复现。
- 如果必须动态验证，只使用授权环境和可清理、低影响动作。

## 可接受验证

| 场景 | 无害证据 |
|---|---|
| 参数进入 `ProcessStartInfo.Arguments` | 构造参数导致工具错误信息变化 |
| 命令输出可回显 | 固定字符串回显，如 `dotnet_audit_probe` |
| 无回显 | 写入授权临时目录固定文件名，再清理 |
| 盲执行 | 授权 DNS/HTTP callback，记录请求 ID |
| 动态代码 | 编译/执行只返回固定字符串，不访问系统资源 |

## 报告字段

- 验证环境。
- 执行身份。
- 输入参数。
- 预期证据。
- 实际证据。
- 清理动作。

## 禁止默认输出

- 删除、加用户、下载执行、横向移动、凭据抓取。
- 针对真实第三方地址的 callback。
- 可复制到未授权系统直接执行的 payload。
